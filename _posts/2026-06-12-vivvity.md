## Overview

Vivvity is an automated processing system for high-throughput DNA and RNA 
analysis on microfluidic chips, developed in collaboration with Revvity as 
part of ongoing research in the Tripathi Lab at Brown University. The device 
is designed to replace manual chip handling workflows and scale into a 
next-generation commercial liquid handler, succeeding the Revvity LabChip 
GxII Touch.

## The Problem

Microfluidic chip-based assays for nucleic acid analysis require precise, 
repeatable fluid handling under controlled pressure conditions. Manual 
operation introduces variability, limits throughput, and creates a barrier 
to scalability. Vivvity addresses this by fully automating chip loading, 
clamping, and pressurization in a compact, programmable platform.

## System Architecture

The device integrates several subsystems:

- **Motion control** — A BTT Octopus v1.1 board running custom firmware 
handles all axis motion, providing the precision positioning needed for 
chip alignment.
- **Electromagnet clamping** — An electromagnet-based clamping mechanism 
secures the microfluidic chip under pressure, verified to hold 30+ psi for 
90+ seconds without failure.
- **Automated pressurization** — Pressure is delivered via a controlled 
input tube, with the system modeled on mechanisms used in existing Revvity 
instrumentation.
- **Electrode interface** — A Revvity-supplied electrode block provides a 
5-pin high-voltage input for fluid manipulation directly on the chip.

## Firmware & Software Development

A significant focus of this project has been firmware development for the 
BTT Octopus v1.1 controller. Beyond standard motion control, I implemented 
custom G-code commands to handle non-motion functions — including 
pressurization sequences and electromagnet control — that fall outside the 
scope of off-the-shelf firmware.

I also developed a professional GUI to replace Pronterface as the 
operator-facing interface, with core functions including axis homing, 
pressurization, and a cleaning mode. The goal was an interface clean and 
simple enough for lab staff with no technical background to operate reliably.

## Status

Active development. The system is currently in firmware integration and GUI 
development phases, with full automated pressurization and motion sequences 
as the next milestones.
